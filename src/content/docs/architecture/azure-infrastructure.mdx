# Azure Infrastructure Overview

Acme Corp runs a shared Azure hub with seven OPCO spokes. The platform standard is **one AKS cluster per OPCO**, shared services in the hub, and **network, identity, and data isolation by design**. The target footprint covers **33 AKS-deployable systems** and keeps **12 non-AKS systems** on legacy or managed platforms until they are ready to move.

## Design principles

1. **Isolation first** — every OPCO gets its own spoke VNet, AKS cluster, namespaces, identities, and private data-plane connectivity.
2. **Shared controls in the hub** — firewall, Bastion, registry, Key Vault, DNS, and monitoring are centralized so spokes consume the same security and operations baseline.
3. **Compliance-aware placement** — PCI workloads run on dedicated node pools and namespaces; Insurance stays as a future-ready landing zone until regulated migrations are approved.
4. **Cost attribution by repo and spoke** — each infrastructure repo maps to one scope, making spend, ownership, and change history traceable.
5. **Private by default** — private AKS APIs, private endpoints for managed data services, and private DNS integration are the standard path.

## System placement summary

| OPCO | Namespace count | AKS-deployable systems | Notes |
|---|---:|---|---|
| FSI | 6 | Core Banking, Payments, Risk Engine, Regulatory, Wealth Mgmt, Data Warehouse | Payments is PCI; Risk Engine uses GPU |
| Retail | 7 | eCommerce, Product Catalogue, Payment, Inventory, Order Fulfillment, Loyalty, Recommendation | Payment is PCI; Recommendation uses GPU |
| Insurance | 0 | None currently scheduled to AKS | Legacy systems remain off-cluster |
| Distribution | 3 | Route Optimisation, Fleet Mgmt, IoT Tracking | Standard workload pool only |
| Telco | 7 | Billing, Rating, CRM, Self-Service, Network Monitoring, Provisioning, Fault Mgmt | Rating workloads use high-CPU pool |
| Media | 7 | CMS, Streaming, Ad Platform, Workflow, Recommendation, Publishing, DRM | Recommendation uses GPU; streaming/ad workloads use general/high-CPU pools |
| Tech | 3 | GitHub Runners, Observability, Infra Platform | Shared engineering platform spoke |
